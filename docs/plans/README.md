# Plans

Use plans for work that spans multiple steps, decisions, or sessions.

## Layout

- `active/`
  - Plans currently being executed (gitignored, local-only)
- `completed/`
  - Finished plans kept for local reference (gitignored, local-only)

Both subdirectories are listed in `.gitignore`. Plans are working notes for the agent and the local developer; they are not part of the published repo.

## Expectations

- Keep plans concise and execution-oriented.
- Track notable decisions and open questions inside the plan.
- Move finished plans out of `active/` promptly.
